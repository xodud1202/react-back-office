## react-back-office 도메인 구조

### 1) 현재 코드 구조
- `app/`: 라우팅과 페이지 진입점을 관리한다.
- `components/`: 관리자 공통 및 화면 컴포넌트를 관리한다.
- `hooks/`: 화면과 기능에서 재사용하는 훅을 관리한다.
- `services/`: 프론트 API 호출과 화면용 서비스 로직을 관리한다.
- `store/`: 전역 상태를 관리한다.
- `types/`, `utils/`: 타입과 공통 유틸을 관리한다.

### 2) 도메인 책임 기준
- 화면, 서비스, 상태, 타입은 가능한 한 동일 도메인 단위로 묶어서 관리한다.
- 검색 결과 리스트는 `ag-grid` 기준으로 설계한다.
- 백엔드 API 추가나 수정이 필요하면 `spring-back-end`에 반영한다.
- 공통 UI는 먼저 `corona-free`에서 재사용 가능 여부를 확인한다.

### 3) 문서 확장 규칙
- 관리자 메뉴나 업무 도메인이 커지면 본 문서 또는 하위 문서로 도메인별 설명을 추가한다.
- 상세 규칙은 `AGENTS.md`가 아니라 `docs/` 아래에 누적한다.
